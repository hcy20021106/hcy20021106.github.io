D²-JSCC

Posted on 2025-08-29

1. 背景

  • JSCC (Joint Source Channel Coding) = 联合信源信道编码。
  • 传统方案：
    • 纯数字传输 (JPEG + LDPC/Polar)
      高 SNR 时接近容量，但低 SNR 时一旦出错会“花屏”。
    • 纯模拟 JSCC
      没有信道编码，失真随 SNR 平滑退化，但高 SNR 浪费容量。
    • 深度 JSCC
      用神经网络直接端到端映射图像→信道符号。鲁棒，但高 SNR 不如数字系统。
  • D²-JSCC（本文提出）：
    结合 深度信源编码 (Deep Source Coding) + 数字信道编码 (Digital Channel Coding)，兼顾鲁棒性与容量利用率。

2. 深度 JSCC vs D²-JSCC

深度 JSCC

  • 没有传统意义上的信道编码。
  • 图像 → CNN 编码 → 信道符号 → 解码器 CNN → 图像恢复。
  • 鲁棒性来自神经网络自动学到的冗余映射。
  • 优点：低 SNR 下平滑退化，不会突然花屏。
  • 缺点：高 SNR 下码率固定，不能榨干信道容量。

D²-JSCC

  • 有传统意义上的信道编码（如 LDPC、Polar）。
  • 流程：
    图像 → 深度信源编码 (DNN) → 比特流
    → 信道编码 (LDPC/Polar) → 符号
    → 无线信道 → 解调/译码
    → 深度信源解码 (DNN) → 重建图像
  • 优点：
  • 低 SNR 下鲁棒（深度信源编码不会像 JPEG 那样产生花屏）；
  • 高 SNR 下接近信道容量（因为仍有传统信道编码）。

3. 为什么 D²-JSCC 更鲁棒

JPEG+LDPC 的问题

  • 熵编码比特流对错误极度敏感。
  • 一旦信道码解码失败 → 整帧报废 → 花屏。

深度信源编码的优势

  • DNN 学到的特征是 分布式冗余，不依赖严格语法。
  • 即使部分比特错误，解码器 CNN 仍能通过上下文恢复合理图像。
  • 结果：低 SNR 下 平滑退化，而不是突然崩溃。

4. 为什么深度 JSCC 高 SNR 下浪费容量

信道容量

  • 高斯信道容量：
    [
    C = \log_2(1+\text{SNR})
    ]
  • 数字系统（LDPC/Polar）：高 SNR 时码率随 SNR 提升，逼近容量。

深度 JSCC 的限制

  • 符号映射是固定的，不会随 SNR 动态提升码率。
  • 优化目标是失真最小化，而不是“码率 = 容量”。
  • 即使噪声趋近于 0，仍存在一定的重建误差。

结果

  • 高 SNR 下，深度 JSCC 的失真有下限，无法无限逼近零失真。
  • 数字系统能充分利用信道容量，而深度 JSCC 会浪费容量。

5. 本文创新点

  1. 架构创新
    提出 D²-JSCC，把深度信源编码和传统数字信道编码结合。

  2. 信源端优化
    使用自适应概率模型（辅助潜变量 z + 条件高斯分布）提升压缩效率。

  3. 理论贡献
    推导了 端到端失真上界 (Theorem 3.1)，把信道误块率和系统参数引入到失真公式中。

  4. 性能优势

  • 低 SNR：比 JPEG+LDPC 鲁棒。
  • 高 SNR：比深度 JSCC 更接近信道容量。
  • 实现了“既鲁棒又高效”的传输。

6. 总结

  • 深度 JSCC：没有信道编码，低 SNR 鲁棒但高 SNR 浪费容量。
  • D²-JSCC：有信道编码，兼顾低 SNR 鲁棒性和高 SNR 容量利用率。
  • 本文的价值在于把深度学习的信源编码与传统通信理论结合，提出了一个 既有实践性能，又有理论保证 的新型通信框架。

熵编码

Posted on 2025-08-29 In 通信原理

1. 基本概念

  • 熵编码（Entropy Coding）是一种无损压缩方法，目的是将符号序列用尽可能少的比特表示。
  • 核心思想来自信息论：
    $H(X) = -\sum_x P(x) \log_2 P(x)$
    • $H(X)$：理论最少平均比特数（香农熵）
    • $P(x)$：符号 $x$ 出现概率
  • 核心原则：
    • 高频符号 → 用较少比特表示
    • 低频符号 → 用较多比特表示

2. 常见熵编码方法

方法 特点 适用场景
霍夫曼编码 (Huffman) 构建前缀码二叉树 符号少，离散概率
算术编码 (Arithmetic) 将整个序列映射到 [0,1) → 二进制 高维连续概率，精细概率分布
范围编码 (Range) 算术编码优化实现 类似算术编码，效率高

Deep Source Encoder 中使用 算术编码。

3. 熵编码在深度源编码中的流程

  1. DNN 特征提取：
    • 输入图像 $x \in X^M$ → DNN → 连续特征向量 $y \in \mathbb{R}^K$
  2. 量化：
    • 连续特征 $y$ → 离散符号 $\tilde{y} \in \mathbb{Z}^K$
  3. 概率建模：
    • 使用侧信息 $\tilde{z}$ 建模条件概率：
      $P_{\tilde{y}|\tilde{z}}(\tilde{y}|\tilde{z})$
    • 使得特征在条件下近似独立，便于编码
  4. 熵编码生成比特流：
    • 对每个量化符号根据概率分配比特长度
    • 高频符号少比特，低频符号多比特
  5. 输出比特流：
    $b = [b_y, b_z]$
    • $b_y$：特征比特
    • $b_z$：侧信息比特

4. 示例

  • 量化后的符号序列：
    $\tilde{y} = [0, 0, 1, 0, 2, 0, 1, 0]$
  • 概率：
    • 0 → 0.625
    • 1 → 0.25
    • 2 → 0.125
  • 熵编码生成比特数：
    • 0 → $\sim 0.678$ bit
    • 1 → 2 bit
    • 2 → 3 bit
  • 平均比特数：
    $B_\text{avg} \approx 10.39$ bit
  • 对比固定长度编码 (3 bit × 8 = 24 bit) → 节省比特

5. 不使用熵编码的情况

  • 固定比特编码：
    • 每个量化符号用相同长度的比特表示
    • 示例：特征范围 0–255 → 每个符号 8 bit
    • 序列 $[3,128,255] \to 00000011\ 10000000\ 11111111$
  • 缺点：
    • 不能利用符号概率 → 压缩效率低
    • 总比特数大于熵编码

6. 总结

  • 熵编码是无损压缩的核心方法，用概率分配比特，理论上最接近熵极限
  • 在 Deep Source Encoder 中：
    1. DNN 提取特征
    2. 量化离散化
    3. 条件概率建模 + 熵编码生成比特流
  • 相比固定比特编码，熵编码显著减少比特数，提高压缩效率

channel coding

Posted on 2025-08-29

1. 基本概念

  • 信道编码 (Channel Coding)：在数字通信中，为了抵抗信道噪声和衰落，将信息比特加冗余形成码字。
  • 目标：
    • 提高抗噪能力
    • 减少误码率
    • 在有限信道资源下尽可能可靠传输

2. 块码 (Block Codes)

  • 描述形式：(N, L) 块码
    • N：信息比特数
    • L：编码比特数（码字长度）
    • 码率：Rc = N / L
  • 功能：
    • 将 N 个信息比特编码成 L 个比特
    • 添加冗余，提高鲁棒性

示例：

  • LDPC 编码

    • 输入：100 个信息比特
    • 输出：200 个编码比特
    • Rc = 100 / 200 = 1/2
    • 输出的 200 个编码比特还属于比特序列，未调制前不能直接发送

  • 卷积码（Convolution Code）
    若卷积码编码（r=1/2），也就是速率r=1/2，输入1bit，输出2bits，等效于在发射端增加冗余，方便接收端做错误检测和纠错。

3. 调制 (Modulation)

  • 将编码比特映射为可在物理信道上发送的信号
  • 例子：16-QAM
    • 每个符号承载 4 个比特
    • 200 个编码比特 → 50 个 QAM 符号
  • 调制后的符号是复数信号点 (I/Q)

4. 端到端流程 (传统数字通信)

5. 通用(N,L)块码理解

  • 论文中常用抽象描述：
    • “通用信道编码器” == 抽象块码模块
    • 可以选择任意具体实现：
      • LDPC
      • Polar Code
      • BCH
  • 输入： N比特
  • 输出： L个比特
  • 调制映射后形成物理符号发送

6. JSCC与深度学习端到端通信

  • 端到端联合源信道编码（JSCC）

    • 不再分开源编码、信道编码和调制
    • 一个深度网络直接学习：
      • 压缩特征
      • 抗噪鲁棒性
      • 可调制映射

  • 例：MDJCM

    • Encoder：输出可在信道直接传输的符号序列
    • Decoder：直接恢复原始数据
    • 不依赖传统信道编码器

7. 笔记总结

  1. 信道编码器是为了在数字传输中增加冗余保护比特
  2. (N, L) 块码是最常用抽象形式
  3. 输出编码比特在调制前仍是比特，调制后成为符号（复数信号）
  4. JSCC/MDJCM 是端到端学习方法，可替代传统源编码 + 信道编码 + 调制流程

JSCC

Posted on 2025-08-28 Edited on 2025-08-29 In 5g

JSCC模型说明

JSCC（Joint Source-Channel Coding）数字联合信源信道编码，特点包括：

  • 端到端训练：支持latent feature -> 调制符号的可微训练
  • 软条件化（Soft Conditioning）：编码器可根据调制阶数（M-QAM）和信道SNR自适应调制特征
  • 噪声近似训练：训练阶段使用连续噪声替代调制/解调操作，使梯度可回传

模型结构

1. 输入信号处理（AnalysisTransform (NTC)）

  • 功能：

    • 将原始图像 [B, C, H, W] 映射为潜在表示 [B, C', H', W']。
    • 降低空间维度，同时提取局部和全局特征。
    • 为 JSCC 编码器提供高效的潜在特征表示。

  • 实现细节：

    • 使用 Swin Transformer 模块和 Patch Embedding 层。
    • 通过多层 Swin Transformer Block 进行特征提取。
    • 输出的潜在特征维度：
      • C'：潜在通道数（例如 256），表示潜在特征维度。
      • H', W'：空间分辨率，经降采样后通常为原图的 1/16。

  • 降维作用：

    • 将高维图像信息压缩为低维潜在特征，减少信道传输冗余。
    • 提取关键特征，增强潜在表示的表达能力。
    • 为后续 JSCC 编码器/信道模块提供适合连续编码的特征表示。

  • 后续处理：

    • 潜在表示 y 可通过 Hyperprior 模块估计尺度参数 (scales_hat) 和均值 (means_hat)。
    • 潜在表示进一步量化为 y_hat，并输入 SynthesisTransform 或 JSCC 编码器进行压缩传输。

2. JSCC编码器（JSCCEncoder）

主要组成：

  • Swin Transformer Blocks：

    • 自注意力机制
      每个patch可访问其他patch的信息，类似冗余信息分布
    • 多层堆叠
      高纬特征融合，增加了特征的鲁棒性
    • MLP变换。
      非线性映射，把信息分布在多个通道-》类似把信号映射到冗余码字空间
      因此刚好满足信道编码的功能：增加冗余、增强抗噪声能力

  • Rate Adaption Layer：根据熵模型和信道需求动态调整编码长度。

    • 根据每个patch的信息量（或者说熵）选择不同的编码率
    • 通过可学习的weight和bias对输入特征进行线性变换，然后用mask限制每个patch的有效长度，实现自适应编码。

  • MergingNet (Modulation Conditional Module, MCM)：

    • 输入：latent feature + 调制阶数 M
    • 对特征进行 软条件化：
    • 作用：让编码器特征适应不同调制阶数

  • AF (Adaptive Feature Layer)：

    • 输入：latent feature + 信道 SNR
    • 对特征进行缩放和平移：
    • 作用：信道条件自适应，提升鲁棒性

  • 输出：经过软条件化和可选调制处理的连续 latent feature

3. 调制近似

  • 将连续latent feature 映射成离散QAM符号
  • 训练阶段使用均匀噪声近似调制

其中M是调制阶数，d是半符号间距，用来保证噪声幅度与符号间距一致

4. 调制与信道

测试阶段

  • 连续latent feature -》 正常QAM调制
  • 经过AWGN或其他信道传输

训练阶段：

  • 使用连续噪声近似调制，使梯度可回传

并且支持不同调制阶数的条件训练

5. JSCC解码器

  • 将收到的信道符号（或连续近似特征）解码为latent feature
  • 可结合hyperprior

理论支持

符号平均功率归一化：

对于 M-QAM（平方型），设符号幅度按标准格排列：

  • 水平/垂直方向的符号坐标:{$-\sqrt M + 1$, $-\sqrt M + 3$, … , $\sqrt M - 1$}.
  • 平均符号功率$P_{avg}$为：$P_{avg} = \frac{2(M-1)}{3}$

噪声近似

  • d对应符号间距的一半
  • 对于均匀量化
    $x_{quantized} ​≈ x_{continuous}​ + u,u ∼ Uniform(−d,d)$
  • 这样训练时梯度可回传，同时模拟调制离散化误差

软条件化

  • 调制阶数和信道SNR作为条件输入
  • 通过线性层 + 非线性激活（Softplus / Sigmoid）调整 latent feature
  • 增强编码器对不同调制阶数和信道条件的适应能力

端到端可微训练

  • 连续噪声替代离散调制，使梯度可回传
  • 后续可结合 STE 或软硬退火进行微调

训练过程

阶段一：source coding training

  • 目标：单独训练基于学习的图像压缩器，使其具备良好的表示能力。
  • 固定参数：冻结 JSC 编码器与解码器 (θf, ϕf)。
  • 训练参数：θg, θh, ϕg, ϕh。
  • 损失函数：
    L(x, x̂, λ) = R + λD(x, x̂)
    其中 R 为比特率，D 为失真 (如 MSE)，λ 为拉格朗日乘子。
  • 说明：这一阶段只关注图像压缩，不涉及信道。

阶段二：JSC encoder和JSC decoder训练

  • 目标：在端到端系统中引入“连续噪声近似调制”，使系统在训练时保持可微，便于联合优化。
  • 初始化：加载阶段一训练好的参数。
  • 训练参数：θg, θh, θf, ϕg, ϕh, ϕf (全模型参与)。
  • 步骤：
    1. 从训练数据中采样图像 x。
    2. 从目标范围内均匀采样 SNR。
    3. 随机选择调制阶数 M。
    4. 根据 M 与功率约束 Es 计算调制间距 d。
    5. 使用 连续噪声近似 通过信道。
  • 损失函数：
    L(x, x̂, λ, M, SNR) = R + λ [ D(x, x̂) + D(x, x̂MD) ]
    其中 x̂ 为压缩器重建图像，x̂MD 为经过端到端 JSCC 模块后的图像。

阶段三：基于 STE 的精调 (STE-based Finetuning)

  • 目标：解决训练与测试阶段之间的不匹配（真实调制是确定性的，不是加噪声），提升模型在实际信道上的性能。
  • 初始化：加载阶段二训练好的参数。
  • 固定参数：冻结发射端参数 (θg, θh, θf)。
  • 训练参数：ϕg, ϕh, ϕf (接收端)。
  • 步骤：
    1. 重复阶段二的数据采样步骤（x, M, SNR, d）。
  • 损失函数：
    与阶段二相同：
    L(x, x̂, λ, M, SNR) = R + λ [ D(x, x̂) + D(x, x̂MD) ]

Notice

其中JSC编码器/解码器是一个神经网络模块，以SNR和mcs为输入，学习如何把特征编码成可传输的信号，再从信号中恢复。

One-Shot Coding

Posted on 2025-08-28 In 5G

基本概念

一次性编码，不需要序列式迭代或多步计算，输出是用向量表示

作用

将离散信息转为神经网络可处理的连续特征。因此one-shot coding后可以用MLP来处理。
类似NLP里的word embedding。
One-shot具有可微性（一对一映射），因此可以进行反向传播。

Notice

信号量化具有不可微性（多对一映射）

Reinforcement Learning

Posted on 2025-08-27 In AI

强化学习概念

  • RL 是一种机器学习方法，通过与环境交互的试错经验学习最优行为（policy），以最大化累计奖励（accumulated reward）
  • 与监督学习不同：RL不需要“正确答案”做训练，而是通过奖励反馈自主学习

RL系统组成：

  • Agent: 学习者和决策者
  • Environment: 智能体作用的外部系统
  • State($s_t$)： 智能体感知到的环境信息
  • Action($a_t$)：智能体在状态下采取的操作
  • Reward($r_{t+1}$)：动作的即时反馈
  • Goal: 找到最佳策略𝜋，实现状态 → 动作的最优映射，使累积奖励最大化。

Q-learning算法

核心思想：通过不断更新动作价值函数Q(s,a)来学习最佳策略
Q值更新公式：
$Q(s_t, a_t) <- (1 - \alpha) * Q(s_t, a_t) + \alpha * [ r_{t+1} + \gamma * max_{a_{t+1}​∈A} * Q(s_{t+1}, a_{t+1}) ]$。

参数说明：

符号 说明
$Q(s_t, a_t)$ 状态 $s_t$ 下执行动作 $a_t$ 的当前 Q 值
$\alpha$ 学习率（0~1），控制新信息更新 Q 值的权重
$r_{t+1}$ 执行动作 $a_t$ 后在下一个状态获得的即时奖励
$\gamma$ 折扣因子（0~1），衡量未来奖励的重要性
$s_{t+1}$ 执行动作后的下一状态
$a_{t+1}$ 下一步选择的动作
$A_{\text{max}}$ 在下一状态下可选择的最大 Q 值动作集合
指示函数，如果 $a_{t+1}$ 是最大动作之一，则为 1，否则为 0

QL-AMC

基于Q-learning的自适应调制编码，用于在无线通信系统中在不同信道条件下动态选择最优MCS

问题背景

传统方法：
固定查表（look-up table）：根据SNR对应的BLER曲线预先生成CQI -> MCS的映射

QL-AMC方案

Q(s,a): 动作-价值函数，表示在动态s下选择动作a的价值
State: 状态集合，在这里状态是CQI（Channel Quality Indicator），即信道质量指标
Action：动作集合，这里是可选的MCS（调制与编码方案）

Notice

Q可以理解为通过RL来推导CQI->MCS的映射，因此不局限于表格的推荐，而是根据环境反馈动态优化。

Reward function:

  • R1 基于BLER的非线形奖励。

    • μ: 每个符号的比特数 (bits per modulation symbol)
    • ν: 编码率 (code rate)
    • BLER: 当前传输块误块率
    • BLER_T: 系统目标 BLER (例如 10% 对 eMBB).

    如果MCS满足BLER目标，奖励等于理论传输速率μ*ν
    否则给惩罚值-1，避免过激MCS

  • R2 基于光谱效率的奖励。

    • 奖励考虑实际成功传输比特数
    • agent 会尝试最大化光谱效率.

说明：

  • Q(s, a) 更新时结合 当前奖励 r 和 未来潜在收益 max_a Q(s’, a’)

  • 最后的Q表的用途是用于决策.

$ a* = argmax_a Q(s, a)$.
也就是对当前CQI s，选择Q值最大的MCS

  • 学习经验
    Q表保留了 RL agent对环境的经验，反映了不同信道条件下选择MCS的最优策略

EVM-based SNR Calculation

Posted on 2025-08-27 In 5G

1. PSD-Based SNR Estimation (ANN Method)

  • 概念: PSD (Power Spectral Density) 表示接收信号功率在频率域的分布。

  • 接收信号 r(t) = s(t) + n(t)，其中 s(t) 是有效信号，n(t) 是噪声。

  • PSD 包含了信号功率和噪声功率。

  • ANN 方法流程:

    1. 接收信号 -> 提取 PSD 特征。
    2. ANN 输入 PSD -> 输出 SNR 分类。
    3. SNR 分类映射到最优 MCS（调制与编码方案）。

  • 优点:

    • 不依赖解调符号或误差向量。
    • 高速移动环境中稳健（PSD 不受多普勒效应显著影响）。
    • 推理计算量低，实时性好。

  • 平均功率计算:

    • 平均功率 $P = ∫ S_x(f) df$（连续信号）或 $P = (1/N) Σ |X[k]|^2$（离散信号）。
    • PSD 积分得到总功率，可用于 SNR 计算: $SNR = P_signal / P_noise$。

2. EVM-Based SNR Estimation

  • EVM (Error Vector Magnitude) 衡量接收符号与理想符号的误差：
    $EVM = sqrt(Σ |r_i - s_i|^2 / Σ |s_i|^2)$
    • $r_i$: 接收符号
    • $s_i$: 理想符号（通常是 pilot/reference symbols）
  • 流程:
    1. 接收 pilot 符号。
    2. 计算误差向量 $e = r_p - s_p$。
    3. 得到 EVM -> 间接推算 SNR -> 映射到 CQI -> 选择 MCS。
  • 局限:
    • 依赖 pilot。
    • 高速移动或多普勒环境下 EVM 会受到星座漂移影响，SNR 估计不稳定。

3. LS-Based SNR Estimation

  • LS (Least Squares) 方法用于信道估计:
    • 基本公式: $h_hat = r_p / s_p$（简单 LS）
    • 信道估计后计算 SNR: $SNR = E[|h s|^2] / E[|r - h s|^2]$
  • 特点:
    • 通过信道增益和噪声功率直接计算 SNR。
    • 也依赖 pilot。
    • 高速移动环境中相对稳健，但需要频繁更新信道估计。

4. 区别总结: EVM vs LS vs PSD+ANN

特性 EVM-based LS-based PSD+ANN
原理 接收符号 vs 理想符号误差 信道估计 + 噪声功率 PSD 特征 + ANN 分类
输入 Pilot / Reference Pilot / Reference 接收信号 PSD
输出 SNR (间接) SNR (直接) SNR 分类
高速移动适应性 差 较好 很好，鲁棒
复杂度 低 中等 低，推理快
使用场景 AMC / CQI映射 信道估计、AMC、功率控制 AMC / CQI映射

5. AMC应用总结

  • CQI驱动 AMC:

    • 基站根据 UE 反馈的 CQI 动态选择 MCS。
    • 高 SNR → 高阶调制 + 高码率 → 高吞吐量。
    • 低 SNR → 低阶调制 + 低码率 → 保证可靠性。
    • 优点: 动态适应信道，频谱效率高。
    • 缺点: 需要反馈，系统复杂。

  • 固定 MCS:

    • 不考虑信道变化，固定使用某个 MCS。
    • 简单，但在差信道下 BLER 高，信道好时未充分利用频谱。
    • 吞吐量和可靠性都不如 CQI 驱动方法。

  • ANN+PSD 方法:

    • 跳过 EVM 或 LS 复杂计算。
    • 利用 PSD 特征直接预测 SNR 分类。
    • 在高速移动信道下鲁棒性更强。
    • 可快速选择最优 MCS，提高系统吞吐量。

6. 结论

  1. PSD 中包含信号和噪声，可以通过积分或求和得到平均功率，从而估计 SNR。
  2. EVM 基于 pilot 符号计算误差向量，间接得到 SNR。
  3. LS 基于 pilot 符号做信道估计，再计算 SNR。
  4. ANN+PSD 方法在高速移动场景下性能优于传统 EVM 方法，计算量低且稳健。
  5. CQI 驱动 AMC 与固定 MCS 的区别在于系统能否动态适应信道变化，前者性能更优。
