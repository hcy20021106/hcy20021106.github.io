信源编码

Posted on 2025-06-07 In 通信原理

Basic Concept

信源编码主要用于提高数据传输的可靠性，纠正通信过程中可能发生的错误。常用的信源编码包括：

  • Reed-Solomon（RS）编码：一种块编码，擅长纠正突发错误。
  • 卷积编码：一种常见的前向纠错码，适用于连续数据流。
  • 交织（Interleaving）：重排数据顺序，扩散突发错误，提高纠错效果。

整体收发流程为:

  1. 发射端
    视频源-》RS编码-》卷积交织-》数据流标记-》
  2. 接收端
    解调后的基带数字信号-》维特比解码-》解交织-》RS解码

Reed-Solomon (RS) 编码

RS编码基本概念

  • RS编码在数据块后添加冗余校验码，能纠正一定数量的错误字节。
  • 典型参数：RS(n, k)，n为编码后数据长度，k为原始数据长度。
  • 纠错能力为 t = (n - k) / 2。

模拟调制

Posted on 2025-06-04 In 通信原理

模拟调制学习记录 📡

记录我对模拟调制技术的学习笔记，分为 AM、DSB、SSB 和 VSB 四种常见调制方式。

📘 AM（Amplitude Modulation）调幅

基本概念

  • 通过改变载波信号的幅度来携带信息信号。
  • 调制信号形式：

$$
s(t) = [A_0 + m(t)] \cos(2\pi f_c t)
$$

其中$A_0\cos(2\pi f_c t)$为载波，$m(t)$为调制信号

特点

  • 简单易实现。
  • 带宽为 (2B)，其中 (B) 为信号带宽。
  • 存在包络检波方式，易于接收。
  • 效率低，载波本身不包含信息但占用功率。

📘 DSB（Double Side Band）双边带调制

基本概念

  • 又称 DSB-SC（Suppressed Carrier）。
  • 将基带信号与载波相乘，抑制直流项。
  • 形式：
    $$
    [
    s(t) = m(t) \cos(2\pi f_c t)
    ]
    $$

特点

  • 没有载波，传输效率高于 AM。
  • 但接收时需要载波同步，复杂度增加。
  • 频谱包含上下两个边带，带宽为 (2B)。

📘 SSB（Single Side Band）单边带调制

基本概念

  • 在 DSB 的基础上，仅保留一个边带（上边带或下边带）。
  • 去除冗余频谱部分，提高频谱效率。

生成方法

  • 滤波法：滤除一个边带。
  • 平衡调制 + 相移法（Hilbert 变换）：生成正交信号组合消除一个边带。

特点

  • 带宽减半，仅为 (B)。
  • 频谱效率高，适合长距离传输（如语音通信）。
  • 实现复杂，载波恢复要求高。

📘 VSB（Vestigial Side Band）残留边带调制

基本概念

  • 在 SSB 基础上，保留一个完整边带 + 另一个边带的一小部分（残留边带）。
  • 实际用于电视视频信号等高带宽场景。

特点

  • 兼顾频谱利用与实现难度。
  • 能较好保留低频成分，解决 SSB 滤波困难问题。
  • 广泛用于模拟电视广播（如 NTSC、PAL）。

📚 小结

类型 带宽 是否包含载波 频谱效率 接收复杂度
AM 2B ✅ 是 低 低（可包络检波）
DSB 2B ❌ 否 中 中（需同步）
SSB B ❌ 否 高 高（需精确滤波/相移）
VSB ≈B~1.5B ❌/部分载波 较高 中高（电视常用）

信道

Posted on 2025-06-04 In 通信原理

定义与分类

  • 狭义信道
    • 有线信道
      同轴电缆，光纤
    • 无线信道
      微波视距传播，卫星中继，移动通信信道
  • 广义信道
    • 调制信道
      调制器输出->解调器输入
    • 编码信道
      编码器输出->译码器输入

调制信道

有一对一/一对多的输入输出端，满足线性叠加原理，对信号有时变和时不变的两种延迟和损耗，其数学模型是叠加有噪声的线性时变/时不变滤波器

恒参信道

对应的信道特性为线性时不变滤波器

FFT

Posted on 2025-05-12 In 通信原理

Basic concept

FFT是在时间复杂度为$\mathcal{O}(n \log n)$完成的离散傅立叶变换（Discrete Fourier transform， DFT）算法，主要应用是加速多项式乘法计算。

Notice

DFT用于有限长的离散信号（通常为有限长度N）

DTFT用于无限长的离散信号（理论上）

DFT用于实际信号处理，如FFT算法

FFT的实际应用

在实际信号收发中，信号在经过调制后输出的复数对是频域信号，这是因为它们表示的是各个子载波上的频域符号，还没有经过IFFT转换为时域波形。通过IFFT后这些复数对才会映射到各个子载波上。例如有四个QPSK符号，对它们进行IFFT：
x[n] = $\frac{1}{4}\sum_{k=0}^{3}e^{j\frac{2{\pi}nk}{4}}$

FFT原理

多项式

A(x) = $\sum_{i=0}^{n-1}{a_i}x^i$ 与 B(x) = $\sum_{i=0}^{n-1}{b_i}x^i$相乘的结果为
C(x) = $\sum_{k=0}^{2n-2}(\sum_{k=i+j}a_ib_j)x^k$
那么这四个点就用来表示这一个OFDM符号对应的四个采样点它们的时域波形。时域信号有以下几个特性：

  • 是4个子载波复数调制符号在频域上的逆变换（IFFT）
  • 保证正交性
  • 可以加循环前缀后发射
  • 接收端做FFT可以完美还原出X[k]

随机过程

Posted on 2025-05-01 Edited on 2025-06-04 In 通信原理

Basic concept

随机过程是随机试验的衍生，具体来说它是Xi(t)的集合，其中Xi是n个变量，理论上n是无强大，Xi的分布函数和t有关，也就是说它的统计学特性受时间影响。

一维分布函数

也就是说只考虑某个固定的时间t，随机变量X(t)的分布函数一般与t有关：
Fx(x,t) = P{X(t)<=x}
当t取遍T中的所有元素时，我们可以得到一系列的分布函数，将这些分布函数的全体所构成的集合称为一维分布函数族，记为
{Fx(x;t)}
这个也就是一维分布函数族,它用来刻画随机过程在各个单独时刻的统计特性。

一维随机过程的n维分布函数族

一维分布函数族刻画了随机过程在各个时刻的统计特征，为了描述随机过程在不同时刻状态之间的统计联系，一般可对任意
个不同的时刻，也就是可以用二维分布函数来表示，如下分布函数所示：

这个分布函数是关于t1 t2 x1 x2四个变量的函数。
例如当t1=1，x1=1；t2=1，x2=1的时候P( x(t=1)<1 , x(t=2)<1 ) 的取值，
对比前面的1维分布函数，后者例如t1=1，x1=1时候，P（ x（t=1）<1）的取值
n维分布函数：为了描述随机过程在不同时刻状态之间的关系，对任意n个不同时刻t1，t2，…,tn引入n维随机变量
（X（t1）， X（t2）， X（t3），…, X（tn））的分布函数

平稳随机过程

平稳随机过程的统计特性不随时间的推移而变化，其一维分布与时间t无关，二维分布只与时间间隔$\tau$有关，如下

  • 一维分布与时间$\tau$无关
    f(x,t) = f(x)
  • 二维分布只与时间间隔$\tau$有关
    f(x1,x2;t1,t2) = f(x1,x2;τ)

平稳随机过程的数字特征

  • 自相关只与时间间隔$\tau$有关，而与时间起点无关
    R(0) = E[ $\epsilon^2(t)$ ] = S
    物理意义： 随机过程的平均功率
  • 均值
    均值与t无关，样本函数围绕一水平线起伏
  • 方差
    方差与t无关，为常数

R($\infty$) = E[$\epsilon$(t)]²
因为时间间隔无限大时，$\epsilon(t)$ 与 $\epsilon(t+\tau)$趋于独立

物理意义： 随机过程的直流功率

平稳随机过程的功率谱密度

假定f(t)为随机过程$\epsilon(t)$的任意实现，对其进行T长度的截断，并在对其进行傅立叶变换。
$\lim_{T \to \infty} \frac{|F_T(w)|^2}{T}$

维纳-辛钦定律
平稳随机过程$\epsilon(t)$的功率谱密度函数$P_\epsilon(w)$和自相关函数R($\tau$)为一对傅立叶变换对

平均功率计算方法:
S = R(0) = E[$\epsilon(t)^2$]
S = $\frac{1}{2\pi}\int_{-\infty}^{\infty}P_\epsilon(w),dw$

高斯随机过程

若随机过程的任意n维分布都是正态分布，则称它为高斯随机过程或正态过程。其n维概率密度函数为：
$f_a(x_1,x_2,…;t_1,…,t_n)$
一维高斯过程的概率密度函数：
X ~ N(a,$\sigma^2$)
$f_1(x)$ = $\frac{1}{\sqrt{(2\pi)}\sigma}exp[-\frac{(x-a)^2}{2\sigma^2}]$
高斯过程在不同时刻取值不相关，则它们也统计独立。
高斯过程经过线性变化（或线性系统）后仍为高斯过程。

正态分布函数

先来看正态概率密度函数
X ～ N(a,$\sigma^2$),
$f_1(x)$ = $\frac{1}{\sqrt{(2\pi)}\sigma}exp[-\frac{(x-a)^2}{2\sigma^2}]$
那么正态分布函数就是将上述函数从负无穷到x上积分
标准/归一化的正态分布是另a=0，$\sigma$=1
$f_1(x) = \frac{1}{\sqrt{(2\pi)}}exp[-\frac{x^2}{2}]$

Q函数

和正态分布的密度分布函数是互补的关系
也就是从x到正无穷积分（正态分布函数是从负无穷到x积分）

误差函数

$\text{erf}(x) = \frac{2}{\sqrt{\pi}} \int_{0}^{x} e^{-t^2} , dt$
互补误差函数和它互补，也就是说是从x到正无穷上积分
误差函数、互补误差函数与正态分布函数存在一个关系

随机过程通过线性系统

有以下几个特性

  • 数学期望
    E[$\epsilon_o(t)$] = aH(0)

  • 自相关函数
    输入平稳随机过程，则输出过程自相关函数只与时间间隔有关

Notice

平稳随机过程通过线性系统，输出也是平稳随机过程

  • 功率谱密度
    $P_o(w)$=$P_i(w)$|$H(w)^2$|
    其中|$H(w)^2$|为功率增益

窄带随机过程

窄带系统要求通带宽度$\Delta f \ll f_c$
窄带过程是指随机过程通过以$f_c$为中心频率的窄带系统的输出过程
$$
\xi(t) = A(t) \cos\big(2\pi f_c t + \phi(t)\big)
$$

  • ( \xi(t) )：窄带随机过程
  • ( A(t) )：随机包络（幅度调制过程）
  • ( f_c )：载波频率（中心频率）
  • ( \phi(t) )：随机相位
通常分析平稳高斯窄带过程$\xi(t)$ 所以均值为0，方差为$\sigma^2$,由于可以写成同向分量$\sigma_c(t)$与正交分量$\sigma_s(t)$线性组合的关系，因此这两个分量的数学期望也均为0，并且二者都平稳（相关函数均只与时间间隔有关），并且二者的自相关函数相同，方差相同，同时也互不相关、统计独立的高斯变量。 简单来讲， 这两个分量为零均值、等方差、不相关、独立的平稳高斯过程。

正弦波加窄带高斯过程

$r(t) = Acos(\omega_c t + \phi) + n(t)$
其中n(t)是均值为0，方差为$\sigma^2$的窄带高斯噪声
然后通过三角函数转换可得r(t)最终也可以分解为同相分量和正交向量，也就是$z_c(t) = Acos{\phi} + n_c(t)$和$z_s(t) = Acos{\phi} + n_s(t)$

随机包络遵循广义瑞利分布，随机相位的概率密度与信噪比环境有关。
对于小信噪比环境，它们分布退化为瑞利分布和均匀分布。
同时它们的整体，也就是正弦波+窄带高斯过程会退化为窄带高斯过程。

白噪声

功率谱密度均匀分布在整个频率范围内的随机过程称为白噪声
由自相关和功率谱密度的傅立叶变换关系可知，白噪声的自相关函数是在$\tau$=0上面有个冲激信号的函数

高斯白噪声

服从高斯分布的白噪声，在任意两个不同时刻上随机变量互不相关，且统计独立。

低通白噪声

功率谱是门函数，对应的自相关函数是sinc函数

带通白噪声

功率是门函数与两个冲激函数的卷积，对应的傅立叶变换是sinc函数与cos函数的乘积

LMMSE base channel estimation

Posted on 2025-04-29 In 5G

Basic Concept

先了解MMSE，MMSE（Minimum Mean Square Error），也就是最小均方误差，是一种常用的统计估计方法，它的目标就是最小化估计值与实际值之间的均方误差（MSE）。在通信系统中，MMSE被广泛用于信道估计，因为它能够在噪声和干扰的影响下，给出一个最优估计。
另外，在信道估计中，交叉相关矩阵通常用于计算数据子载波和导频子载波之间的相关性，这对于估计信道的状态至关重要。

C语言实现线程池

Posted on 2025-04-28 Edited on 2025-05-27 In C

Basic Concept

在高并发程序中，频繁创建和销毁线程会造成较大的资源消耗和系统开销。线程池（Thread Pool）是一种常用的并发编程技术，通过维护一个线程集合来处理任务，避免频繁创建和销毁线程，提高系统性能
线程池通常具备以下功能：

  • 支持动态添加任务
  • 线程复用，减少线程创建销毁的开销
  • 任务队列支持FIFO机制
  • 线程池的销毁与资源回收

Producer-Consumer Model

以下元素和结构几乎出现在所有经典线程池实现中:

组件 功能说明 是否为标准做法
线程数组 预先创建固定数量的线程，用于处理任务 ✅ 是
任务队列 存放尚未处理的任务，按顺序排队 ✅ 是
互斥锁（mutex） 用于保护任务队列的访问，防止多个线程竞争资源 ✅ 是
条件变量（cond） 当任务队列为空时阻塞线程，有新任务时唤醒工作线程 ✅ 是
无限循环的 worker_thread 每个线程不断尝试从任务队列中取任务并执行 ✅ 是
stop 标志 控制线程池的销毁流程，优雅地通知线程退出 ✅ 是
任务结构包含 function + arg 支持传入任意函数和参数，使线程池任务更通用 ✅ 是

这涉及同步的概念，需要满足同步与互斥，因此需要使用互斥锁（pthread_mutex_t）来保证任务队列操作的线程安全，以及需要条件变量（pthread_cond_t）用于通知线程处理任务
