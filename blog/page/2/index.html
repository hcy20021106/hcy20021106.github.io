Diffusion 模型体系对比：DDPM · Inpainting · CDDM

本 README 对比三类扩散模型：

  1. DDPM（标准扩散模型）
  2. Stable Diffusion Inpainting（mask concat 条件）
  3. CDDM（Channel Denoising Diffusion Model）

1. DDPM —— 标准扩散模型

Forward（逐步加噪）

DDPM 的前向加噪公式为：

$$x_t = \sqrt{\bar{\alpha}_t}, x_0 + \sqrt{1 - \bar{\alpha}_t}, \epsilon,\quad \epsilon \sim \mathcal{N}(0,I)$$

特点：

  • 按 $t=1\dots T$（$T=1000$）逐步加噪
  • 使干净的 $x_0$ 越来越像纯噪声

Reverse（逐步去噪）

模型预测噪声：

$$\epsilon_\theta(x_t, t)$$

反向采样：

$$x_{t-1} = f(x_t, \epsilon_\theta, t)$$

特点：

  • 需要很多步（50–1000）
  • 可以带文本条件（cross-attention）

2. CDDM —— Channel Denoising Diffusion Model

CDDM 用于无线信道去噪，目标是回复语义特征x0

2.1 forward(类比信道噪声)

CDDM定义的forward加噪声：

$x_t = \sqrt{\bar{\alpha}_t}, x_0 + \sqrt{1 - \bar{\alpha}_t}, W_n \epsilon$
其中：

$W_n$：由 Rayleigh fading + AWGN 构成
$\epsilon \sim \mathcal{N}(0,I)$
实际信道模型：
$y_r = \frac{1}{1+\sigma^2}, W_s x + \frac{\sigma}{1+\sigma^2}, W_n \epsilon$

关键区别：

DDPM：数学加噪，多步
CDDM：真实信道噪声，一步即可覆盖所有衰落 + 噪声机制

2.2 Reverse

初始化：
$x_m = y_r$
每步计算：
$z = W_n * \epsilon_\theta(x_t, h_r, t)$
更新：
$x_{t-1} = \sqrt{\bar{\alpha}_{t-1}} \left( \frac{x_t - \sqrt{1 - \bar{\alpha}_t}, z}{\sqrt{\bar{\alpha}t}} \right) + \sqrt{1 - \bar{\alpha}{t-1}}, z$
特点：

只需 $m\approx 5$–$20$ 步
包含信道增益 $h_r$ 的条件输入

2.3 Loss

$L_{\text{CDDM}}(\theta) = \mathbb{E}{x_0,\epsilon,h_r,t} \left[ | \epsilon - \epsilon\theta(x_t, h_r, t) |^2 \right]$
和 DDPM 类似，但输入多了 $h_r$， forward 噪声来自真实信道。

3. Stable Diffusion Inpainting —— Mask 作为 concat 条件

UNet 输入是多个条件张量的拼接：

名称 含义
x_t 当前带噪 latent
mask 1=要修补，0=保留
masked_image 提供上下文内容

模型预测：
$\epsilon_\theta(x_t,; mask,; masked_image,; t)$
特点：

仍然是 DDPM 的多步去噪
concat 是 Inpainting 的核心机制
用于修补图像，不是通信信道

项目 DDPM Inpainting（concat） CDDM
Forward 多步高斯噪声 多步高斯噪声 Rayleigh + AWGN 信道噪声
Reverse 50–1000 步 50–1000 步 $m$ 步（5–20）
条件 文本 文本 + mask + context 信道增益 $h_r$
输入 $x_t$ concat([...]) $x_t, h_r$
输出 图像 修补图像 语义特征（feature）

Diffusion-based Denoising & Feature Imputing (Semantic Communication)

本 README 用于理解 diffusion 在语义通信/抗噪中的 denoising 原理，包括 forward / reverse、训练 / 推理、UNet 输入、mask、噪声差异等内容。

1. Core Idea

训练：学习从噪声恢复语义 latent
推理：对 corrupted latent 执行 conditional reverse denoising

一句话：
Train with synthetic noise; infer on corrupted latent.

2. Forward diffusion (training)

对完整 latent y0 加噪，得到 y_t：

$y_t = sqrt(alpha_t) * y0 + sqrt(1 - alpha_t) * eps_t$

其中 eps_t ~ N(0, I)

随着 t 增大，噪声增加，最终：

y_T ~ N(0, I)

目的：

  • 构造 noisy latent
  • 训练噪声预测器（UNet）

特点：

  • forward 只在训练执行
  • 不使用 mask
  • 不使用 corrupted latent y_hat

3. Reverse diffusion (inference)

推理阶段执行逐步去噪：

eps_hat = UNet(y_tilde_t, t, y_hat, M)

更新：
y_tilde_{t-1} = reverse_update(y_tilde_t, eps_hat)

最终输出恢复后的 y_tilde_0

特点：

  • reverse 只在推理执行
  • 包含 conditional 信息（y_hat 和 mask M）

4. UNet inputs

Training:
UNet(y_t, t)

Inference:
UNet(y_tilde_t, t, y_hat, M)

说明：
y_tilde_t = 当前 noisy latent (推理)
y_hat = 破损 latent（信道输出）
M = mask，表示哪些位置缺失
t = diffusion timestep

5. Mask M

M 用于指示哪些 latent 位置丢失需要补全：

M = 1 表示该位置需要生成
M = 0 表示该位置保持接收端值

通常做 dilation 得到 perceptive mask，使恢复区域更平滑。

6. Forward noise != Channel noise

forward噪声（人为添加）与信道噪声（真实丢包/衰落）无关。

Diffusion 学习的是语义恢复能力，而不是匹配具体噪声分布。

7. Initialization during inference

推理起点不是随机噪声，而是 corrupted latent：

y_tilde_T = y_hat

8. Loss per timestep?

训练时不需要对所有 t 计算 loss，只随机采样一个 t：

loss = || eps - eps_theta ||^2

这是 DDPM / DDIM 的标准训练方式。

9. Training pseudocode

for each sample:
y0 = encoder(x)
sample t
y_t = sqrt(alpha_t)*y0 + sqrt(1-alpha_t)*eps
eps_hat = UNet(y_t, t)
loss = ||eps - eps_hat||^2
optimizer.step()

10. Inference pseudocode

y_tilde_T = y_hat
for t = T … 1:
eps_hat = UNet(y_tilde_t, t, y_hat, M)
y_tilde_{t-1} = reverse_update(y_tilde_t, eps_hat)

output = y_tilde_0

11. Key understanding

  • forward = synthetic corruption (training only)
  • reverse = actual recovery (inference only)
  • UNet learns denoising
  • mask guides inpainting
  • inference uses corrupted latent as start

12. One-sentence summary

Diffusion learns semantic restoration from noisy latent during training and performs conditional reverse denoising on corrupted latent with masks during inference.

13. Misconceptions

误区：训练噪声等于信道噪声 → 错
误区：推理从高斯开始 → 错
误区：reverse 在训练执行 → 错
误区：每个 t 都计算 loss → 错

真相：

  • forward only for training
  • reverse only for inference
  • sample one timestep per update

VAE

如果将VQ理解为直接将编码向量聚类为K个不同的类别，那么FSQ就是将编码向量归纳出d个属性，每个属性划分为L个等级，从而直接表达了$L^d$
个不同的整数。

不管是从参数量、几何直观或者表达能力来看，其实FSQ都不如VQ，但为什么FSQ有机会取得比VQ更好的结果呢？笔者认为有两方面的原因。

第一个原因，是encoder和decoder太强。虽然FSQ本身弱一些，但是encoder和decoder都足够强了，所以基于神经网络的万能拟合能力假设，FSQ相对于VQ的劣势，完全可以在encoder和decoder中弥补过来。

第二个原因，是VQ的“队友”（梯度）太弱。VQ的经典问题是编码表坍缩：当编码表增大时，编码表并没有被充分利用起来，反而由于恶性竞争导致编码表聚集到一块了，经典表现就是一个5000的编码表，最终效果还不如500的编码表。归根结底，这是梯度不够合理所致，尽管VQ已经巧妙地设计了梯度，但对于argmin
这种硬指派的运算，基于梯度的优化都存在“赢者通吃”问题，这是坍缩的根本原因，而FSQ的Round
运算并不涉及到指派，它是直接取的近似值.
