channel coding

Posted on 2025-08-29

1. 基本概念

  • 信道编码 (Channel Coding)：在数字通信中，为了抵抗信道噪声和衰落，将信息比特加冗余形成码字。
  • 目标：
    • 提高抗噪能力
    • 减少误码率
    • 在有限信道资源下尽可能可靠传输

2. 块码 (Block Codes)

  • 描述形式：(N, L) 块码
    • N：信息比特数
    • L：编码比特数（码字长度）
    • 码率：Rc = N / L
  • 功能：
    • 将 N 个信息比特编码成 L 个比特
    • 添加冗余，提高鲁棒性

示例：

  • LDPC 编码

    • 输入：100 个信息比特
    • 输出：200 个编码比特
    • Rc = 100 / 200 = 1/2
    • 输出的 200 个编码比特还属于比特序列，未调制前不能直接发送

  • 卷积码（Convolution Code）
    若卷积码编码（r=1/2），也就是速率r=1/2，输入1bit，输出2bits，等效于在发射端增加冗余，方便接收端做错误检测和纠错。

3. 调制 (Modulation)

  • 将编码比特映射为可在物理信道上发送的信号
  • 例子：16-QAM
    • 每个符号承载 4 个比特
    • 200 个编码比特 → 50 个 QAM 符号
  • 调制后的符号是复数信号点 (I/Q)

4. 端到端流程 (传统数字通信)

5. 通用(N,L)块码理解

  • 论文中常用抽象描述：
    • “通用信道编码器” == 抽象块码模块
    • 可以选择任意具体实现：
      • LDPC
      • Polar Code
      • BCH
  • 输入： N比特
  • 输出： L个比特
  • 调制映射后形成物理符号发送

6. JSCC与深度学习端到端通信

  • 端到端联合源信道编码（JSCC）

    • 不再分开源编码、信道编码和调制
    • 一个深度网络直接学习：
      • 压缩特征
      • 抗噪鲁棒性
      • 可调制映射

  • 例：MDJCM

    • Encoder：输出可在信道直接传输的符号序列
    • Decoder：直接恢复原始数据
    • 不依赖传统信道编码器

7. 笔记总结

  1. 信道编码器是为了在数字传输中增加冗余保护比特
  2. (N, L) 块码是最常用抽象形式
  3. 输出编码比特在调制前仍是比特，调制后成为符号（复数信号）
  4. JSCC/MDJCM 是端到端学习方法，可替代传统源编码 + 信道编码 + 调制流程
